Introduction

Un peu d'histoire

Bell Labs

  • Fondé en 1925 par AT&T
  • 9 prix Nobels & 4 prix Turing
  • Mecque du développement technologique au 20e siècle

S

  • Inventé par John Chambers au sein des Bell Labs
  • Développé pour la programmation statistique
  • Objectif: “To turn ideas into software, quickly and faithfully”
  • Inventé dans la seconde moitié des années '70
  • John Chambers est Canadien! Né à Toronto en 1941
  • Auparavant, Fortran était utilisé pour la programmation statistique
  • Emphase sur le temps d'implémentation, pas le temps de calcul
  • A rapport avec le niveau du langage

Concepts de base 🤓

Niveaux de langage

  • Compromis entre convivialité et performance
  • Facile pour nous vs Facile pour l'ordinateur


machine ⮕ assembleur ⮕ bas niveau ⮕ haut niveau
Examples:
Assembleur
ARM, x86, MIPS
Bas niveau
C, Pascal, ALGOL
Haut niveau
R, Python, C++, Java
Exemple: Calcul d'un nombre de Fibonacci sur x86 (source): 

                    8B542408 83FA0077 06B80000
                    0000C383 FA027706 B8010000
                    00C353BB 01000000 B9010000
                    008D0419 83FA0376 078BD989
                    C14AEBF1 5BC3
Exemple: Calcul de la suite de Fibonacci en assembleur x86-64 (en partie, source): 

                    xor %edx, %edx
                    mov $10, %ebx
                    div %ebx
                    addb $48, %dl
                    movb %dl, (%esp, %ecx, 1)
                    dec %ecx
                    cmp $0, %ax
                    je .done_pl
                    jmp .loop_pl
Exemple: Calcul de la suite de Fibonacci en Haskell: 

                    fib :: Integer -> Integer
                    fib 0 = 0
                    fib 1 = 1
                    fib n = fib (n - 1) + fib (n - 2)
Langage machine
  • Directement exécuté par la machine
  • Très difficile à comprendre pour un humain
  • Dépend directement de la machine sur laquelle on travaille
Langage de haut niveau
  • Éloigné du langage machine, n'a presque rien à voir avec celui-ci
  • Conçu pour faciliter la vie du programmeur
  • Relativement indépendant de l'architecture

Compilation vs interprétation

  • Ordinateur exécute uniquement du code machine
  • Tout autre langage doit être «traduit»: compilation
  • Autre possibilité: faire exécuter le code par un logiciel auxiliaire


R est un tel logiciel auxiliaire!
  • Logiciel auxiliaire = interpréteur

  • R est multiparadigme
    • Impératif
    • Fonctionnel

    • Array programming

R en bref

R est:
  • Un langage de haut niveau
  • Un langage interprété
  • Un langage multiparadigme
  • Une implémentation de S
  • R vise à être beaucoup plus près du langage mathématique que du langage machine
  • 2 premiers points: en lien avec l'objectif original de R (“To turn ideas into software, quickly and faithfully”)
  • R, c'est S tout comme CPython est Python!

Git et GitHub 😱

  • Git: système de gestion des versions
  • Git \( \neq \) GitHub
  • Glosaire et commandes importantes
    • git clone
    • git add
    • git commit
    • git pull
    • git push
    • git remote
  • Git sert à gérer les différentes versions d'un groupe de fichiers
  • GitHub permet de stocker des copies de dépôts à distance
  • GitHub est une peu comme Dropbox mais plus structuré

Dépôt Git

  • “Repository”, “repo”
  • Conteneur pour un ensemble de fichiers, par exemple
    • Répertoire local
    • Dépôt GitHub
  • Clône \( := \) copie d'un dépôt
  • Cas typique
    • Un dépôt sur GitHub
    • Un clône local

Workflow typique

  1. Mettre à jour le dépôt local: git pull
  2. Modifier le dépôt local 💻
  3. Commettre les modifications: git commit
  4. Rincer et répéter 🚿
  5. Envoyer les modifications commises vers GitHub: git push
